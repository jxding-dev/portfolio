PROJECT STORY

문제 정의 & 해결 방향

어떤 문제를 발견하고, 어떻게 풀었는지

CHALLENGE

설계 전 마주한 문제들

  • 고객이 직원 없이 직접 주문하는 환경에서 UX가 복잡하면 이탈 발생
  • 사이즈·온도·추가 옵션 조합이 많아 실시간 금액 계산 로직 구현 필요
  • 장바구니 중복 담기·수량 변경·취소 등 상태 관리가 복잡
  • POS 화면은 관리자가 빠르게 확인·처리할 수 있는 별도 구조 필요
SOLUTION

설계 & 구현으로 해결한 방법

  • 카테고리 → 상품 → 옵션 → 장바구니 3단계 플로우로 주문 흐름 단순화
  • 옵션 선택 즉시 금액을 재계산하는 실시간 계산기 로직 구현
  • 상품 중복 체크 예외처리 및 장바구니 동적 상태 관리 구현
  • Figma로 POS 대시보드 UI를 별도 설계하여 관리 효율 극대화
KEY FEATURES

핵심 기능 & 구현 포인트

기술적으로 가장 집중한 부분들

🧾

카테고리 동적 렌더링

메뉴 데이터를 JS 배열·객체로 구조화하고, 카테고리 탭 전환 시 해당 항목을 DOM에 동적으로 렌더링합니다.

💰

실시간 금액 계산

사이즈·온도·추가 옵션 선택마다 이벤트 리스너가 작동해 최종 금액을 즉시 갱신합니다. 장바구니 합계도 자동 반영됩니다.

🛒

장바구니 상태 관리

동일 상품 중복 담기 예외처리, 수량 변경, 개별·전체 삭제를 구현하고 빈 장바구니 시 안내 UI를 조건부 렌더링합니다.

📊

POS 관리자 UI 설계

Figma로 대시보드·통계·테이블 현황·메뉴 관리 화면을 설계. 실시간 주문 수락 및 매출 시각화 구조를 기획했습니다.

📱

반응형 레이아웃

모바일·태블릿 환경에 맞는 터치 친화적 UI를 CSS 미디어쿼리로 구현해 다양한 디바이스에서 안정적으로 동작합니다.

🎨

UIUX 화면 설계

고객 동선을 분석해 주문 흐름을 최소 단계로 설계하고, 포토샵·피그마로 브랜드 무드에 맞는 시각 언어를 정립했습니다.
